# TOOLS.md - 工具使用指南

## 🔧 工具选择策略

你有两类工具可用：**轻量工具** 和 **重型工具**。

### 轻量工具（优先使用）

| 工具 | 用途 |
|------|------|
| `read` | 读取单个文件 |
| `write` | 创建/覆盖文件 |
| `edit` | 修改文件中的特定文本 |
| `exec` | 执行命令 |
| `list` | 列出目录内容 |
| `grep` | 搜索文件内容 |
| `search` | 搜索网络获取实时信息 |
| `research` | 深度研究一个主题 |

**适用场景**：
- 查看文件内容
- 简单的单文件修改
- 运行单个命令
- 快速查询
- 搜索网络获取实时信息
- 深度研究某个主题

---

## 🔍 网络搜索

### `search` — 快速搜索

**判断标准：** 问自己 "我真的知道吗？"

**何时搜索**：
- 训练数据截止后可能变化的信息
- 实时数据（股价、天气、比分）
- 你有任何不确定
- 用户问"最新"、"现在"、"今天"

### `research` — 深度研究

当需要全面了解一个主题（多角度分析、带引用来源）时，使用 `research` 而不是多次 `search`。

## 🤖 重型工具：`claude_code`

**这是一个强大的子代理工具**，可以自主完成复杂的多步骤编程任务。

**何时使用 `claude_code`**：

✅ **必须使用的场景**：
- 调试复杂 bug（需要读多个文件、分析、修复、测试）
- 实现新功能（涉及多个文件的创建/修改）
- 重构代码（跨文件的重构）
- 编写测试用例（需要理解代码结构）
- 项目初始化或配置（如设置 eslint、typescript 等）

❌ **不要使用的场景**：
- 简单的单文件查看/修改
- 运行单个命令
- 查询天气/时间等简单信息
- 回答问题（不涉及代码操作）

**使用示例**：

```
用户："帮我调试 src/utils.ts 里的排序函数"
→ 使用 claude_code，因为需要：读文件、分析问题、修复、验证

用户："查看 package.json"
→ 使用 read，简单任务

用户："帮我在项目里添加一个用户认证模块"
→ 使用 claude_code，因为是多步骤功能实现

用户："把第 10 行的 foo 改成 bar"
→ 使用 edit，简单修改
```

**调用方式**：
```json
{
  "task": "详细描述任务目标，包括文件路径、预期结果、验证方式"
}
```
